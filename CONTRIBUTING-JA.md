# コントリビューションガイドライン

`improvie` プロジェクトへのご関心ありがとうございます！このガイドラインでは、プロジェクトに貢献する方法について説明します。

## 🛠 コントリビューションの種類

以下の方法でプロジェクトに貢献できます：

1. **バグ報告**
   アプリケーションの問題を見つけた場合は、[GitHub Issues](https://github.com/improvie/improvie/issues) に報告してください。

2. **機能提案**
   新しい機能や改善案がある場合は、[GitHub Issues](https://github.com/improvie/improvie/issues) で提案してください。

3. **コードの改善**
   バグ修正や新機能の実装を行い、プルリクエストを送信してください。

4. **ドキュメントの改善**
   ドキュメントの誤字修正や内容の追加も歓迎します。

5. **テストの追加**
   テストケースを追加して、アプリケーションの品質向上に貢献してください。

## 🛠 コーディングスタイル

- **JavaScript/TypeScript (Svelte)**
  - `eslint` を使用してコードスタイルをチェックします。`bun run lint` を実行し、`bun run lint:fix` で自動修正を行ってください。
  - 型チェックには `svelte-check` を使用します。`bun run check` を実行して確認してください。

- **Rust**
  - `rustfmt` を使用してコードスタイルを整えます。`cargo fmt` で自動整形を行ってください。
  - `cargo clippy` を使用してコードのリンティングを行います。

- **エディタ設定**
  - `.editorconfig` によって自動的にエディタの設定が適用されます。

## 📚 ドキュメント

- 機能の追加や仕様の変更を行った場合は、関連するドキュメントを必ず更新・追加してください。
- ドキュメントは明確で簡潔に記述し、必要に応じてサンプルコードを含めてください。

## 🧪 テスト

- 変更をプッシュする前に、すべてのテストが成功することを確認してください。
  - `src` 以下では `bun test` を使用してテストを実行します。
  - `src-tauri` 以下では `cargo ci` を使用してテストを実行します。

## 🔄 Git & GitHub

- コミットメッセージは、変更内容を明確に示すように記述してください。
- プルリクエストを作成する際は、関連するIssueをリンクし、変更の目的を明確に説明してください。
