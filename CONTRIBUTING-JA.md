# コントリビューションガイドライン

`improvie` プロジェクトへのご関心ありがとうございます！このガイドラインでは、プロジェクトに貢献する方法について説明します。

## 🛠 コントリビューションの種類

以下の方法でプロジェクトに貢献できます：

1. **バグ報告**
   アプリケーションの問題を見つけた場合は、[GitHub Issues](https://github.com/improvie/improvie/issues) に報告してください。

2. **機能提案**
   新しい機能や改善案がある場合は、[GitHub Issues](https://github.com/improvie/improvie/issues) で提案してください。

3. **コードの改善**
   バグ修正や新機能の実装を行い、プルリクエストを送信してください。

4. **ドキュメントの改善**
   ドキュメントの誤字修正や内容の追加も歓迎します。

5. **テストの追加**
   テストケースを追加して、アプリケーションの品質向上に貢献してください。

## 🧪 コーディングガイドライン

- **コードスタイル**
  プロジェクトのコードスタイルに従ってください。`cargo fmt` や `cargo clippy` などのフォーマッタ・リンターの使用を推奨します。

- **ドキュメント**
  機能の追加や仕様の変更を行った場合は、関連するドキュメントも必ず更新・追加してください。

- **テスト**
  既存のコードを変更したり、新機能を追加した際は、必要に応じてテストを追加または更新してください。
  変更をプッシュする前に、すべてのテストが成功することを確認してください：

  ```bash
  cargo ci
  ```
